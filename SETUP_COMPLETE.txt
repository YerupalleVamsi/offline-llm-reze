## 🔧 Advanced Options

### Use Different Port
Edit `RUN_UNIFIED.bat`:
```batch
python -m uvicorn app.main:app --host 127.0.0.1 --port 8001
```

### Development (Separate Ports)
If you want frontend dev server + backend separately:

Terminal 1 (Backend on 8000):
```bash
cd backend
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Terminal 2 (Frontend dev on 5173):
```bash
cd frontend
VITE_API_URL=http://127.0.0.1:8000 npm run dev
```

Then access: `http://localhost:5173`

## ❓ Common Questions

**Q: Where are the frontend files served from?**
A: From `frontend/dist/` folder (created by `npm run build`)

**Q: Why do I need to build frontend?**
A: The build creates optimized production files. The script does this automatically.

**Q: Can I develop without building?**
A: Yes, see "Development (Separate Ports)" above

**Q: What if I close the terminal?**
A: The server stops. Just run `RUN_UNIFIED.bat` again.

**Q: Can I use port 8000 for something else?**
A: Not while this app is running. Change the port in the script or close the other app.

**Q: Do I need FFmpeg?**
A: Only if you want audio processing. Without it, audio features won't work.

## 🆘 If Something Goes Wrong

### Check 1: Is Backend Running?
```
Visit: http://localhost:8000/health
Expected: {"status":"ok","message":"..."}
If: Page doesn't load → Backend not running
```

### Check 2: Are Dependencies Installed?
```
Windows Terminal (in backend folder):
pip install -r requirements.txt

Windows Terminal (in frontend folder):
npm install
```

### Check 3: Frontend Built?
```
If frontend/dist/ folder doesn't exist:
cd frontend
npm run build
```

### Check 4: Check Logs
Terminal shows errors? Read them carefully:
- "Module not found" → Missing dependency (npm install or pip install)
- "Port in use" → Port 8000 in use (change port or kill other app)
- "FFmpeg not found" → Install FFmpeg for audio

## 📞 Support

All errors should be visible in the terminal where you ran the script.

1. **Read the error message** - It usually tells you exactly what's wrong
2. **Check the documentation files** - They cover common issues
3. **Look at the logs** - Terminal shows detailed information

If you see "Backend not responding", the issue is usually:
- Backend process crashed (check errors in terminal)
- Port 8000 in use (close other apps or use different port)
- Frontend not built (run `npm run build` in frontend folder)
