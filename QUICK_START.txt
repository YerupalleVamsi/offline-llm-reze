# QUICK START - Single Port (8000) Setup

## 🚀 For Windows Users

Simply **double-click** this file:
```
RUN_UNIFIED.bat
```

That's it! You'll see:
- Console showing build progress
- Backend starting
- Message: "Frontend UI: http://localhost:8000"

**Then open browser:** `http://localhost:8000`

---

## 🚀 For Linux/Mac Users

Run this command:
```bash
bash run-unified.sh
```

Then open browser: `http://localhost:8000`

---

## What's Happening

1. **Builds the frontend** (creates optimized production build)
2. **Starts the backend** on port 8000
3. **Backend serves the frontend** at `http://localhost:8000`
4. **API endpoints** at `http://localhost:8000/api/*`

Everything runs on **ONE port** - no more "failed to fetch" errors!

---

## ✅ Verification

Once running, you should see:

**In the terminal:**


**In your browser at http://localhost:8000:**
- Chat interface loads

## If Something Goes Wrong

### "Command not found" (Linux/Mac)
```bash
chmod +x run-unified.sh
bash run-unified.sh
```

### "Port 8000 already in use"
Another app is using port 8000. Kill it:

**Windows (PowerShell as admin):**
```powershell
Get-Process -Id (Get-NetTCPConnection -LocalPort 8000 -ErrorAction SilentlyContinue).OwningProcess | Stop-Process -Force
```

**Linux/Mac:**
```bash
lsof -i :8000 | grep -v PID | awk '{print $2}' | xargs kill -9
```

### "npm: command not found"
Install Node.js from https://nodejs.org/

### "python: command not found"
Install Python 3.10+ from https://python.org/

### "FFmpeg not found" (Audio won't work)
- **Windows:** Download from https://ffmpeg.org/download.html, add to PATH
- **Mac:** `brew install ffmpeg`
- **Linux:** `apt-get install ffmpeg`

## Need Help?

Check the logs in the terminal window. Most errors show exactly what's wrong:
- Missing dependencies → install with pip or npm
- Port in use → kill the other process
- FFmpeg missing → install it
- Build errors → check Node.js version

All output is logged to help diagnose issues!
